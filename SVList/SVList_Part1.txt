Part1 总体架构设计
1.1 系统目标

SVList 是一个：

基于 UGUI ScrollRect 的高性能虚拟列表框架。

核心目标：

百万级数据支持

动态高度

多模板

0GC滑动

60FPS

可扩展Layout

支持异步创建

支持对象池

支持JumpToIndex

支持Editor可视化配置

原则：

Core 与 MonoBehaviour 分离

算法层：

不依赖 Unity。

方便：

单元测试
未来 UIToolkit 迁移
ECS 迁移

1.7 状态机

整个列表：

{
    Uninitialized,

    Initializing,

    Running,

    Refreshing,

    Disposed
}

防止：

初始化过程中：

Refresh()

导致空引用。

1.8 Item状态机

每个Renderer：

enum ItemState
{
    Pooled,

    Active,

    PendingRecycle
}
Pooled

对象池内。

不可见。

Active

正在显示。

PendingRecycle

等待回收。

避免：

Recycle()

Recycle()

重复进入池。

1.11 核心原则
不Destroy

只Recycle。

不Instantiate

走Scheduler。

不全量Refresh

走Diff。

不线性查找

走FenwickTree。

不Update轮询

走事件驱动。

不让Renderer参与算法

Renderer只负责：

Bind()

Unbind()
