5.4 为什么要 LateUpdate

如果：

容易：

LateUpdate：

这是非常重要的。

5.9 Item状态机

对象池内。

不可见。

显示中。

等待回收。

因为：

Recycle(item)

5.10 CreateScheduler

类：

内部：

count=0

{

    CreateOne();

    count++;
}

5.12 RecycleScheduler

流程：

while(queue.Count>0)
{
    item.Renderer.OnUnbind();

    item.Rect.SetParent(poolRoot);

    item.State=Pooled;

    pool.Release();
}

注意：

再：

否则：

事件监听泄漏。

5.14 Renderer接口

推荐：

interface ISVItemRenderer<T>
{
    void OnCreate();

    void OnBind(T data,int index);

    void OnUnbind();

    void OnRecycle();
}

作用：

OnCreate

缓存组件。

只执行一次。

GetComponent<TMP_Text>()

避免重复。

OnBind

刷新数据。

OnUnbind

取消监听。

停止协程。

OnRecycle

彻底销毁时。

释放资源。
